# 🌐 LAN Implementation using Hub and Switch

## 🎯 Objectives
- Implement LAN using hub and switch  
- Observe communication between multiple PCs  

## 🛠️ Software
- Cisco Packet Tracer

## 📖 Introduction
LAN connects devices within a limited area.  
- Hub → Broadcasts to all ports  
- Switch → Uses MAC table for intelligent forwarding  

## ⚙️ Procedure
1. Multiple PCs were added  
2. Hub and switch were connected separately  
3. Straight-through cables were used  
4. IPs assigned in same subnet  
5. Connectivity tested using `ping`  

## 🌐 IP Configuration
| PC   | IP Address     |
|------|---------------|
| PC1  | 192.168.1.2   |
| PC2  | 192.168.1.3   |
| PC3  | 192.168.1.4   |

Subnet Mask: `255.255.255.0`

## ⚡ Hub vs Switch
- Switch uses MAC address table  
- Hub broadcasts blindly  
- Switch reduces traffic and collisions  

## 📌 Key Insights
- LAN implemented successfully  
- Hub caused broadcast traffic  
- Switch improved efficiency  
- Selective data transmission observed
